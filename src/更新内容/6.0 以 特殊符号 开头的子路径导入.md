# 以 #/ 开头的子路径导入

这是值是支持了 `#/` 开头的子路径导入。 也就是说 `tsconfig.json` 中 `paths` 字段中的路径还是要配置的 只是可以以 `#/` 开头。 


当 Node.js 增加对模块的支持时，它新增了一项名为 [“子路径导入”](https://nodejs.org/api/packages.html#subpath-imports) 的功能。这本质上是 一个名为 [imports](https://nodejs.org/api/packages.html#imports) 的字段，允许包为其自身包内的模块创建内部别名。


```json
{
  "name": "my-package",
  "type": "module",
  "imports": {
    "#root/*": "./dist/*"
  }
}
```

这使得 `my-package` 中的模块可以从以 `#root/` 开头的路径导入：

```javascript
import * as utils from "#root/utils.js";
```

而不是使用如下的相对路径：

```javascript
import * as utils from "../../utils.js";
```

这个功能有一个小麻烦，就是开发者在指定子路径导入时，必须在 `#` 后面写一些内容。这里我们用了 `root`，但它其实没什么用，因为除了
`./dist/` 之外，我们没有映射其他目录。

使用过打包工具的开发者也习惯使用路径映射来避免冗长的相对路径。打包工具的一个常见约定是使用简单的 `@/` 作为前缀。遗憾的是，子路径导入
**完全不能以 `#/` 开头**，这让试图在项目中采用子路径导入的开发者感到十分困惑。

但最近，Node.js 从 **`#/` 开始支持子路径导入**。这使得包可以使用简单的 `#/` 前缀来进行子路径导入，而无需额外添加一个段。

```json
{
  "name": "my-package",
  "type": "module",
  "imports": {
    "#/*": "./dist/*"
  }
}
```

这一点在较新的 **Node.js 20** 版本中得到支持，因此 TypeScript 现在在 `nodenext` 和 `bundler` 选项下为
`--moduleResolution` 配置提供了支持。
